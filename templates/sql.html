SQL Injection

To atak polegający na wstrzyknięciu złośliwego kodu SQL do zapytania, które aplikacja wysyła do bazy danych. Jeśli dane od użytkownika (np. login lub ID) są bezpośrednio wstawiane do zapytania SQL bez walidacji, atakujący może zmodyfikować działanie zapytania.

Spróbuj zalogować się, wpisując dowolny login i hasło (login: admin hasło: admin123 albo login: user hasło: user123).

Jeżeli wpiszesz login ' OR 1=1 -- i dowolne hasło, to zostaniesz zalogowany bez podania prawidłowych danych, ponieważ warunek 1=1 jest zawsze prawdziwy — to klasyczny przykład ataku SQL Injection.
